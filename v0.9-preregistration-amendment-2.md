## 1. Reason for Amendment

The initial v0.9 batch (140 runs) produced three findings:

- **Design A**: full survival across all six cost levels. Zero hazard entries in 60 experimental runs. The pre-wired aversion bias sustains hazard avoidance independently of architectural impassability.
- **Design C**: partial learning. 29/60 runs achieved zero Phase 3 entries; 31/60 runs failed to achieve Phase 3 cleanness within the 20,000-step run length.
- **Architectural robustness**: the full v0.8 developmental architecture (concentration, attractor ratio, mastery, individuation) survives removal of hazard-handling components, holding stable at v0.8-equivalent levels across all four hazard modes.

The Design C partial-learning result raises a specific developmental question: do the 31/60 failed runs reflect *slow* learning that would resolve given more time, *plateau* behaviour where extended time produces no further convergence, or *degraded* learning where extended time actively worsens avoidance? Each interpretation has distinct implications for what the architecture actually does when deprived of its pre-wired aversion scaffolding.

This is a question about learning dynamics over time, not about cost levels. It is not addressed by any extension to the cost range, and it is not addressed by the initial batch. It requires a manipulation of run length, holding all other parameters constant. This amendment specifies that manipulation and the decision tree for interpreting its results.

The initial batch's findings are complete as-is and do not depend on the extended-run-length experiment. This amendment adds a follow-up investigation to an established result rather than modifying it.

## 2. Research Question

Does extended run length enable Design C learners to achieve Phase 3 cleanness in runs that failed within 20,000 steps, and if so, is the learning process best characterised as slow convergence, plateau, or degradation under extended time?

## 3. Experimental Design

**3.1 Scope.** Design C runs only: 60 experimental runs (`hazard_mode = cost_no_aversion`, six cost levels × 10 runs each, using the same pre-registered cost levels as the initial batch) plus 10 control runs (`hazard_mode = impassable_no_aversion`). Design A is excluded; its full survival result at 20,000 steps is not informative about learning dynamics.

**3.2 Manipulation.** The single variable changed from the initial batch is `NUM_STEPS`. Cost function, hazard mechanism, cost levels, run count per condition, threshold-crossing definition, and all agent-architecture parameters remain unchanged.

**3.3 Batch 1 (80,000 steps).** Quadruple the original run length. Phase durations scale proportionally under the existing phase-transition logic: Phase 1 unchanged at the prescribed path length (~331 steps), Phase 2 extends from ~11,669 to ~47,669 steps, Phase 3 extends from 8,000 to 32,000 steps. 70 runs total. Approximate compute time: 12 minutes on reference hardware.

**3.4 Batch 2 (160,000 steps).** Eightfold the original run length, executed conditionally based on Batch 1 results (see Section 5). Phase 2 extends to ~95,669 steps, Phase 3 to 64,000 steps. 70 runs total. Approximate compute time: 24 minutes.

**3.5 Execution sequence.** Batch 1 is run first. Batch 2 is run in the same session, immediately after Batch 1 completes, if the Batch 1 outcome triggers it under the decision tree in Section 5. No interpretive judgement is made between batches; the decision to run Batch 2 is mechanical from the Batch 1 Phase 3 cleanness count.

## 4. Primary Metric

**Phase 3 cleanness count**: the number of Design C experimental runs (out of 60) with zero hazard entries during Phase 3. This is the Design-C-relevant metric committed in the first amendment and reported alongside the total-entries metric in the initial batch.

Secondary metrics captured but not used for decision-tree triggering:
- Total hazard entries per run (for comparison with initial batch distributions)
- Total cost incurred per run (for understanding learning expense)
- Phase 2 vs Phase 3 entry distribution (for temporal learning pattern)
- Full set of v0.8/v0.9 developmental metrics (concentration, attractor ratio, mastery, individuation) — to confirm the developmental architecture remains intact at extended run length.

## 5. Pre-Specified Decision Tree

The Batch 1 outcome is classified into one of three categories using pre-specified Phase 3 cleanness thresholds. Each category determines both the interpretation and the next action.

**5.1 Category A — Substantially higher cleanness.** Phase 3 cleanness ≥ 48/60 (80%+).

*Interpretation.* Extended run length resolves the partial-learning signature observed at 20,000 steps. The 31/60 failed runs in the initial batch were insufficient-time artefacts; given four times the run length, the architecture learns avoidance reliably. The Design C finding is therefore: *the developmental architecture can learn hazard avoidance from experienced cost alone, but requires substantially more time than the pre-wired alternative.*

*Next action.* Conclude the extended-run-length investigation. No Batch 2 executed. Finding reported as: "Design C achieves Phase 3 cleanness in ≥80% of runs at 80,000 steps, compared to 48% at 20,000 steps, supporting slow-convergence interpretation."

**5.2 Category B — Similar cleanness.** Phase 3 cleanness between 22/60 and 36/60 inclusive (within ±7 runs of the initial 29/60).

*Interpretation.* Learning has plateaued. The 31/60 failed runs are structurally different and will not converge within reasonable additional time. This is itself a finding: *the architecture exhibits bimodal learning outcomes — some biographical trajectories successfully internalise cost-based avoidance, others do not, and the distinction does not resolve with more time.* Category B requires confirmation that the plateau is genuine and not a slower asymptote that would eventually converge if given still more time.

*Next action.* Execute Batch 2 (160,000 steps) in the same session as confirmation. Two possible confirmations:
- If Batch 2 produces Phase 3 cleanness also within the Category B range (22–36/60), the plateau is confirmed. Finding reported as: "Design C exhibits bimodal learning outcomes that do not resolve with extended run length up to 160,000 steps."
- If Batch 2 unexpectedly produces Category A results (≥48/60), the plateau was not genuine and the learning horizon is longer than 80,000 steps but within 160,000 steps. Finding reported with appropriate caveat about the non-monotonic convergence profile.
- If Batch 2 produces Category C results (≤21/60), see cross-category note below.

**5.3 Category C — Substantially lower cleanness.** Phase 3 cleanness ≤ 21/60.

*Interpretation.* Extended run length actively degrades learning relative to the 20,000-step baseline. Candidate mechanism: preference accumulation over very long horizons produces preference landscapes strong enough to override cost-based Q-value penalties — the agent "prefers" certain cells so intensely that hazard costs cannot dislodge the preference. This would be a meaningful finding about the preference drive's behaviour over extended time, distinct from the original internalisation question.

*Next action.* Conclude the extended-run-length investigation for v0.9. No Batch 2 executed. Finding reported as: "Design C learning degrades under extended run length, suggesting preference-drive dynamics interact adversely with time in the absence of pre-wired aversion." This outcome opens a separate investigative line — preference-drive dynamics over extended time — that is reserved for a v0.10 research phase and not pursued within v0.9.

**5.4 Cross-category note.** If Batch 2 produces a result in a different category from Batch 1 (other than the specific Category B → Category A case noted above), the combined finding is reported with full transparency about the non-monotonic profile across run lengths. No further batches are run in v0.9; any such pattern opens a separate investigative line for v0.10.

## 6. What Is *Not* Being Tested

Design A is not rerun at extended run length. Its full survival result at 20,000 steps is complete as-is; extended run length adds no information to a zero-entry result.

The cost range is not modified. The pre-registered cost levels (0.1, 0.5, 1.0, 2.0, 5.0, 10.0) are used unchanged.

The threshold-crossing definition is not modified. Phase 3 cleanness is reported using the same zero-entry criterion as in the initial batch.

No further amendments to the Design A or Design C architectures are introduced. The hazard mechanism, cost function, aversion bias treatment, and all agent parameters remain exactly as specified in the first amendment.

## 7. Reporting Commitment

All runs from Batch 1, and from Batch 2 if executed, will be reported regardless of outcome. The decision-tree category assignment and the resulting action (run Batch 2 or conclude) will be stated before the results are presented, not inferred from them. Per-run data will be released as a CSV alongside the initial batch's CSV. The finding associated with the triggered category will be reported using the category's pre-specified language.

Negative, null, and unexpected results will be reported with the same prominence as expected results. A Category C outcome (learning degradation under extended time) will be reported as fully and prominently as a Category A outcome (successful learning with time).
